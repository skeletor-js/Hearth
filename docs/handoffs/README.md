# Handoffs

Task-specific continuation notes live here. Use this folder when work is interrupted or another agent/human needs to pick up a specific thread.

Use one dated file per handoff:

```text
YYYY-MM-DD-short-task-name.md
```

Each handoff should include:

- task / desired outcome
- source of truth: plan, issue, PR, Proof doc, transcript, or file path
- what is already done
- exact next action
- files, sources, or systems touched
- blockers / decisions needed
- suggested skills or tools for the next agent

Do not use this as a general state log, diary, or archive. Durable engineering learnings belong in `docs/knowledge/` or `docs/solutions/`; intended work belongs in `docs/plans/` or repo-native issue/PR systems.
